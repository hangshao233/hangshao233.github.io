CTFHub-web(Git泄露)

发表于2021-02-04|更新于2021-02-04|Web
|字数总计:334|阅读时长:1分钟|阅读量:

当前大量开发人员使用git进行版本控制，对站点自动部署。如果配置不当,可能会将.git文件夹直接部署到线上环境。这就引起了git泄露漏洞。

如果你没有掌握基本的 git 命令：
https://blog.csdn.net/weixin_45254208/article/details/107810572

Log

1. 使用dirsearch扫描，发现git泄露

2. 使用GItHack获取源码
这里要确保是python2版本

如果事先没切换成python2，可以将python替换为python2或者py -2试一下

3. GitHack目录下多了一个dist目录

4. 进入到dist目录下的git环境

5. 使用git log查看提交历史
发现上一个版本叫add flag，于是回滚到上一个版本

6. 版本回滚

生成了包含flag的文件夹

7. get flag

Stash

以同样的方法将源码clone下来

解法一：

1. 执行 git stash list 发现 stash

2. 执行 git stash pop 发现从 git 栈中弹出来一个包含flag的文件

解法二：

使用cat .git.refs/stash打开stash文件，然后执行git diff比较工作区和暂存区

index

1. 同上，clone源码

2. flag直接就出来了
难道是因为题目叫 index ，所以在“首页”就直接有答案了？
