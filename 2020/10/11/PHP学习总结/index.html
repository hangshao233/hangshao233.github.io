PHP学习总结

布尔类型的转换

https://www.php.net/manual/zh/language.types.boolean.php

当转换为 boolean 时，以下值被认为是 FALSE:

  • 布尔值 FALSE 自身
  • 整型值 0 (零)
  • 浮点型值 0.0 (零)
  • 空字符串, 以及字符串 “0”
  • 不包括任何元素的数组
  • NULL，uninitialized
  • 从空标记生成的 SimpleXML 对象

超全局变量

  • $GLOBALS：全局作用域中的全部可用变量
  • $_SERVER：服务器和执行环境的信息
  • $_GET：通过 GET 方法提交的数据
  • $_POST：通过 POST 方法提交的数据
  • $_COOKIE：通过 HTTP Cookies 方式传递给当前脚本的变量所组成的数组
  • $_REQUEST：包含了 $_GET，$_POST 和 $_COOKIE 的所有信息
  • $_FILES：通过 POST 方式上传到服务器的文件数据
  • $_ENV：通过环境方式传递给当前脚本的变量组成的数组
  • $_SESSION：当前脚本可用 SESSION 变量组成的数组

global

PHP中，全局变量在函数内部是不可见的，可通过global或$GLOBALS来访问

  • global实为一个新指针
  • 若有同名，则初始化会与同名变量指向同一数据；若无同名，则初始化为null
  • 不可以在定义global变量的时候直接赋值

static

  • 进入函数的时候静态变量被创建，执行到初始化语句时被初始化
  • 定义静态变量时若不写初始化的值，则默认初始化为 null

运算符

  • $a == ​$b
    • 会进行类型转换
    • 值相等，则返回 TRUE，否则返回 FALSE
  • $a === ​$b
    • 不会进行类型转换
    • 值相等，类型相等，则返回 TRUE，否则返回 FALSE

单引号、双引号、Heredoc

  • 单引号

    • 只支持\‘和\\
    • 速度快

  • 双引号

    • 变量替换
    • 支持更多转义符
    • 速度慢

Heredoc：

https://www.php.net/manual/zh/language.types.string.php

https://wiki.php.net/rfc/flexible_heredoc_nowdoc_syntaxes

  • 变量替换
  • 起始标识符后不能有其它字符
  • 结束标识符缩进不能大于内容中任何一行的缩进

（PHP 7.3 以前）：

  • 结束标识符需顶格
  • 结束标识符分号后不能有其它字符
  • 结束标识符分号后需换行

字符串

字符串长度

strlen()

返回值：字符串长度

mb_strlen()

  • $encoding：字符编码。如果省略，则使用内部字符编码

返回值：字符串长度

字符串查找

strpos()

查找字符串中某部分字符串首次出现的位置，区分大小写

  • $haystack：在哪个字符串中查找
  • $needle：要查找的字符串，如果needle不是一个字符串，那么它将被转换为整型并被视为字符的顺序值（ASCII）
  • $offset：指定从 haystack 中的哪个字符开始查找， 如果是负数，搜索会从字符串结尾指定字符数开始。返回的位置数字值仍然相对于 haystack 的起始位置

返回值：找到，返回位置；未找到，返回false

stripos()

查找字符串中某部分字符串首次出现的位置，不区分大小写

返回值：找到，返回位置；未找到，返回false

strrpos()

查找字符串中某部分字符串最后出现的位置，区分大小写

  • $offset：指定从 haystack 中的哪个字符开始查找，如果是负数的偏移量，将会导致查找在字符串结尾处开始的计数位置处结束。返回的位置数字值仍然相对于 haystack 的起始位置

返回值：找到，返回位置；未找到，返回false

strripos()

查找字符串中某部分字符串最后出现的位置，不区分大小写

  • $offset：指定从 haystack 中的哪个字符开始查找，如果是负数的偏移量，将会导致查找在字符串结尾处开始的计数位置处结束。返回的位置数字值仍然相对于 haystack 的起始位置

返回值：找到，返回位置；未找到，返回false

字符串替换

str_replace()

如果 search和replace为数组，那么str_replace()将对subject做二者的映射替换。

如果replace的值的个数少于search的个数，多余的替换将使用空字符串来进行。

如果search是一个数组而replace是一个字符串，那么search中每个元素的替换将始终使用这个字符串

  • $subject：执行函数的数组或字符串。如果subject是一个数组，替换操作将遍历整个subject，返回值也将是一个数组
  • $search：查找的目标值。区分大小写。数组可以指定多个目标
  • $replace：替换值。数组可以被用来指定多重替换
  • $count：替换发生的次数

返回值：返回替换后的数组或字符串

str_ireplace()

$search不区分大小写

substr_replace()

https://www.php.net/manual/zh/function.substr-replace.php

  • $string：执行替换的数组或字符串。如果string是一个数组，替换操作将遍历整个string，返回值也将是一个数组
  • $start：如果start为非负数，替换将从string的start位置开始；如果start为负数，替换将从string的倒数第start个位置开始
  • $length：
    • 正数，表示string中被替换的子字符串的长度
    • 负数，表示待替换的子字符串结尾处距离string末端的字符个数
    • 0，FALSE或NULL，将replacement插入到string的start位置处
    • 不提供此参数，默认到末尾
  • $replacement：替换值。数组可以被用来指定多重替换

返回值：返回替换后的数组或字符串

字符串截取

substr()

  • $string：待截取的字符串

  • $start：如果start为非负数，截取将从string的start位置开始；如果start为负数，截取将从string的倒数第start个位置开始；如果string的长度小于start，将返回FALSE

  • $length：

    • 正数，表示string中被截取的子字符串的长度
    • 负数，表示待截取的子字符串结尾处距离string末端的字符个数
    • 0，FALSE或NULL，返回一个空字符串
    • 不提供此参数，默认到末尾

返回值：返回截取的子串或false

mb_substr()

  • $str：待截取的字符串
  • $start：如果start为非负数，截取将从string的start位置开始；如果start为负数，截取将从string的倒数第start个位置开始
  • $length：
    • 正数，表示string中被截取的子字符串的长度
    • 负数，表示待截取的子字符串结尾处距离string末端的字符个数
    • 0，FALSE，返回一个空字符串
    • NULL或不提供此参数，默认到末尾
  • $encoding：字符编码。如果省略，则使用内部字符编码

返回值：返回截取的子串

函数

  • PHP中可以在函数定义之后调用，也可以在函数定义之前调用
  • PHP不支持函数重载
  • 默认参数必须从最后一个参数开始从右往左写，不间断

返回引用

要想返回引用需要在函数定义和调用时都加上&

返回引用时，就真的返回了该变量的引用，没有新建临时变量；返回值时，新建了临时变量

排序函数

函数 说明
sort() 对数组元素进行升序排序
rsort() 对数组元素进行降序排序
asort() 对数组元素进行升序排序，并保持索引关系
arsort() 对数组元素进行降序排序，并保持索引关系
ksort() 按照数组的键值对数组进行升序排序，并保持索引关系

natsort()相当于asort($array,SORT_NATURAL);

以sort()为例

sort()不保持索引关系，会从0开始赋予新的键名

参数说明如下：

  • $array：要排序的数组
  • $sort_flags：为可选参数，默认为SORT_REGULAR，用来定义函数排序的模式。有如下值：
    • 0 或 SORT_REGULAR
      • 正常排序
    • 1 或 SORT_NUMERIC
      • 将数组元素当作数值进行比较，但不会真的改变数据类型
      • 不能当作数值的元素不参与排序，会被放置在最前
    • 2 或 SORT_STRING
      • 将数组元素当作字符串进行比较
    • 3 或 SORT_LOCALE_STRING
    • 4 或 SORT_NATURAL
      • 对于数值：正常排序，会将数组元素当作数值进行比较，但不会真的改变数据类型
      • 对于字符串：非数值部分按字符串排序，数值部分整体按数值排序
      • 数值<字符串
    • 5 或 SORT_FLAG_CASE
      • 不区分大小写排序字符串
      • 能与SORT_STRING或 SORT_NATURAL 一起用，通过|连接

PHP正则操作

https://www.php.net/manual/zh/book.pcre.php

分隔符

当使用 PCRE 函数的时候，模式需要由分隔符闭合包裹。分隔符可以是任意非字母、数字、反斜线、空白字符。

经常使用的分隔符是正斜线(/)、hash符号(#) 以及取反符号(~)

preg_match()

执行正则表达式匹配

  • $pattern：模式，即正则表达式
  • $subject：输入字符串
  • $matches：将被填充为搜索结果。$matches[0]将包含整个模式匹配到的文本，$matches[1]将包含第一个捕获子组匹配到的文本
  • $flags：
    • 0：默认
    • PREG_OFFSET_CAPTURE：保存偏移量。如果传递了这个标记，会改变填充到matches参数的数组，使其每个元素成为一个由第0个元素是匹配到的字符串，第1个元素是该匹配字符串在输入字符串中的偏移量
  • $offset：指定从哪个位置开始搜索

返回值：返回匹配次数。它的值将是0或1或false。preg_match()在第一次匹配后将会停止搜索。如果发生错误会返回false

preg_match_all()

执行全局正则表达式匹配

  • $flags：
    • 不能同时使用PREG_PATTERN_ORDER和PREG_SET_ORDER
    • PREG_PATTERN_ORDER：结果排序为$matches[0]保存整个模式的所有匹配, $matches[1] 保存第一个子组的所有匹配，以此类推
    • PREG_SET_ORDER：结果排序为$matches[0]包含第一次匹配得到的所有匹配(包含子组)，$matches[1]是包含第二次匹配到的所有匹配(包含子组)的数组，以此类推
    • PREG_OFFSET_CAPTURE：保存偏移量

返回值：返回匹配次数，如果发生错误返回FALSE

preg_grep()

在数组中搜索匹配模式的元素

  • $input：输入数组
  • $flags：
    • 0：默认
    • PREG_GREP_INVERT：函数将返回输入数组中与给定模式pattern不匹配的元素组成的数组

返回值：返回与模式匹配/不匹配的元素组成的数组，键还是原来的键

preg_replace()

替换

https://www.php.net/manual/zh/function.preg-replace.php

  • $pattern：要搜索的模式。可以是一个字符串或数组
  • $subject：输入字符串或数组
  • $replacement：用于替换的字符串或数组
  • $limit：每个模式在每个subject上进行替换的最大次数。默认是 -1(无限)
  • $count：如果指定，将会被填充为完成的替换次数

返回值：返回替换后的字符串或数组，发生错误返回null

preg_filter()

https://www.php.net/manual/zh/function.preg-filter.php

只返回匹配成功的结果，其余同preg_replace()

preg_split()

通过一个正则表达式分隔字符串

  • $limit：如果指定，将限制分隔得到的子串最多只有limit个，返回的最后一个子串将包含所有剩余部分。limit值为-1， 0或null时都代表”不限制”
  • $flags：
    • PREG_SPLIT_NO_EMPTY：如果这个标记被设置，函数将仅返回分隔后的非空部分
    • PREG_SPLIT_DELIM_CAPTURE：如果设置了此标志，模式中的括号表达式也将被捕获并返回
    • PREG_SPLIT_OFFSET_CAPTURE：附加偏移量

返回值：返回分隔后得到的子串组成的数组，或者在失败时返回FALSE

数据库操作

mysqli_connect()

返回值：成功返回对象，失败返回FALSE

mysqli_select_db()

返回值：成功返回true，失败返回FALSE

mysqli_query()

https://www.php.net/manual/zh/mysqli.query.php

  • $resultmode：可选参数，用来修改函数的行为。可以是下列值的任意一个
    • MYSQLI_USE_RESULT（如果需要查询大量数据，使用这个）
    • MYSQLI_STORE_RESULT（默认值）

返回值：失败时返回 FALSE，通过mysqli_query() 成功执行SELECT, SHOW, DESCRIBE或 EXPLAIN查询会返回一个mysqli_result 对象，其他查询则返回TRUE

mysqli_fetch_row()

该函数的每一次后续调用都将返回结果集中的下一行

返回值：以索引数组的形式返回结果集中的一行，在没有更多行的时候返回null

mysqli_fetch_assoc()

https://www.php.net/manual/zh/mysqli-result.fetch-assoc.php

该函数的每一次后续调用都将返回结果集中的下一行

返回值：

以关联数组的形式返回结果集中的一行，键为字段名；在没有更多行的时候返回null

如果字段名有重复，则最后一个字段会是优先的

mysqli_fetch_array()

该函数的每一次后续调用都将返回结果集中的下一行

  • $resulttype
    • MYSQLI_NUM：返回索引数组
    • MYSQLI_ASSOC：返回关联数组
    • MYSQLI_BOTH：兼而有之

返回值：以索引数组或关联数组或二者兼有的形式返回结果集中的一行；在没有更多行的时候返回null

返回关联数组，如果字段名有重复，则最后一个字段会是优先的

mysqli_fetch_object()

mysqli_fetch_all()

$resulttype

  • MYSQLI_NUM：返回索引数组
  • MYSQLI_ASSOC：返回关联数组
  • MYSQLI_BOTH：兼而有之

返回值：以索引数组或关联数组或二者兼有的形式返回整个结果集

返回关联数组，如果字段名有重复，则最后一个字段会是优先的

mysqli_close

返回值：成功返回true，失败返回false

mysqli_num_rows()

返回值：返回结果集的行数。

如果返回的行数大于 PHP_INI_MAX，则行数将以字符串的形式返回

杂项

trim()

过滤字符串首尾处的空白字符（或者其他字符）

  • $character_mask：过滤字符。可以使用 “*..*” 列出一个字符范围。

    如果不指定该参数，trim()将去除这些字符：

    • “ “：普通空格符
    • “\t”：制表符
    • “\n”：换行符
    • “\r”：回车符
    • “\0”：空字节符
    • “\x0B”：垂直制表符

返回值：过滤后的字符串

**ltrim()**：过滤字符串开头的空白字符（或者其他字符）

**rtrim()**：过滤字符串末端的空白字符（或者其他字符）

md5()

  • $str：输入字符串
  • $raw_output：
    • FALSE：返回32位十六进制数
    • TRUE：返回16字节二进制数

返回值：32位十六进制数的字符串或16字节二进制数所代表的字符串

getenv()

获取服务器和执行环境的信息，有点像$_SERVER

  • $varname：变量名
  • $local_only：为真将只返回本地环境变量

返回值：返回环境变量$varname的值，如果$varname不存在则只返回false

$_SERVER没有以下3个参数：https://blog.csdn.net/jerryvd/article/details/88847406

HTTP_X_FORWARDED_FOR；HTTP_VIA；HTTP_CLIENT_IP

PHP中获取客户端ip地址getenv(“REMOTE_ADDR”) 与$_SERVER[‘REMOTE_ADDR’]的区别：

getenv(“REMOTE_ADDR”) 在 apache下能正常获取ip地址，而在iis中，不起作用；

$_SERVER[‘REMOTE_ADDR’]可在apache中成功获取客户端ip地址，在iis下也同样有效
