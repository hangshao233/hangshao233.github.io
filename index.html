来谈谈我对英语的看法

今天突然想聊一聊我对英语和英语学习的看法，当然了，聊英语的过程中，会说一些其他的内容，主线是英语，其他内容围绕英语和英语学习展开。

以前一直想把这种内心的感受写成博客发出来，毕竟博客本来就是多元化的，既能传递技术，又能表达情感，还能分享生活，今天刚好有这个机会，就来写一写我现在的所思所想。

其实以我目前的水平，写不出那种技术性的、传授性的博客，我的博客更多的是一个笔记本，使得自己在遗忘了一些内容的时候，可以回去复习一下。但是我也很希望有人能从我的笔记里学到东西，因此我一直都是尽量把我的博客写得更容易理解一些。我相信有一天，我的博客还是能够帮助到一些人的。

今天，是我第一次分享学习以外的东西，分享的这个东西，就是文章开头提到的英语和英语学习。

为什么我会突然地想到这个呢？因为最近不是软考嘛（文章发出来之后软考应该就过去了，我自然是考不上的，但是不影响咱发感想呀0.0），软考复习的过程中，有一个题目是这样的：

这个我没了解过，但是我看到了一个选项，说的是目录"/proc"用于存放进程和系统信息，我当即就选择了这一个选项，为什么呢？因为进程的英文就是process，那这个目录名称很明显就是process的缩写了。这个题，我不会，但是对了，是我所学习的英语帮助我答对了这一题，因此我想聊一聊英语，正好可以开始我的第一篇非技术性博客。

前段时间在B站上把郑强教授的视频基本上都看了一遍，2005年的视频，到现在依然受用，他当时的很多话，甚至成功地预言了2020年的今天。在他的视频里面，他提到过一个话题，他当时表示，外语是一门工具，学了外语不要到那个母语国家去混，因为到了那儿你就没有价值了，只有当外语作为你和你本国语言的桥梁时，你的价值才得以体现。郑强教授还说他有的时候就不想教了，教了之后，很多学生都跑到外国去了，说不准有一天中国跟美国干起来了，美国打过来的导弹里面就有他的学生做的高分子橡胶圈。郑强教授忧心教育，居安思危，有血性有担当，敢于说真话，有强烈的爱国精神，他认为学了英语之后，把英语作为工具建设自己的国家，这样才有价值，虽然这句话不能说绝对正确，但是我对此表示敬佩，因为郑强讲的爱国我是认同的，他讲”外语是一门工具“我也是认同的。

美国比我们发达（目前是如此），在很多领域是美国人奠定基础，虽然很多方面中国已经赶超了，但是很多标准是美国制定的，比如编程语言要用英文写，国际通用语言是英语，计算机很多其他的方面也是美国人定下的标准，在这些领域，我们学习英语只是学习一门工具而已。在这一点上，我跟郑强教授想的一样，学了英语，应该用于建设自己的国家，而不是去讲英语的国家去混。

聊一聊我从小到大的英语学习历程吧：

高三，偶然看了一部日本的励志电影，这部电影一定程度上激励了我，电影的女主人公，啥都不会，但是她能从零学起，在楼梯上，在房间里，她能贴满写着知识点的便利贴，最后考上日本的庆义大学，据说这个大学在日本很牛B。当时这个电影很大程度上激励了我，我也是从那之后下定决心把英语学好的，也是从那时开始恶补英语和化学的，化学这里就不提了0.0。

不学英语一时爽，恶补英语火葬场啊。正所谓”因上努力，果上随缘“，同样有一句话也很适用，那就是”因上堕落，果上落魄“。为了弥补，高三我每天坚持做英语阅读，整个高三一年，除了过年那几天没有做，剩下的日子我一天不落。开始我很多单词不认识，不认识我就查字典，相信很多人会觉得那么多单词都不认识，一个一个查字典岂不是太浪费时间了？但是我所想的是，有那个犹豫的功夫，早就查了好些单词了。渐渐地，我做阅读，从经常查字典，到基本不怎么查；从先查再做，到做了再查；从完全不懂，到理解大意；从艰难做完，到养成习惯；从觉得英语很无聊，到觉得部分英语短文还挺有意思。对了，我还像那个日本励志电影里面的女主一样，把房间的一面墙上贴了很多便利贴，上面都是分类、对比之后的单词组，每天午饭后和睡前都会复习。最终高考的时候，我把80分的英语水平提高到了120分。

英语这个东西，我当时学着学着就喜欢上了，一个是因为学英语只需要记忆，不太需要动脑子，这是高三学习中的一种脑力放松，二是因为你能看懂英语短文之后，里面很多故事还是挺有意思的。反正我高三的后半年确实喜欢上了英语，我自己都搞不明白，就是学英语很有热情，原本我是极度不喜欢英语的。

然而进入大学之后，英语就被我遗忘了，或者说是刻意放弃了，但每个学期的英语考试还是能以65分到75分的成绩通过的。我们学校大一又不能考英语四级，所以大二我们寝室集体考英语四级，四个人都是从进大学就没学英语的人，结果自然是全军覆没了。第二次考四级的时候，我和一个室友都涨了100多分过了四级，另一个室友继续不复习所以没过，还有一个室友准备比赛去了，所以第二次四级就没没去考，当然他也是有很大收获的，一个人带队拿了网安比赛的省赛一等奖，这位大佬就是我博客的友情链接里面的那位“松松”。

备战考研的阶段很快就要到来了，英语仍然是一门工具，希望这个工具能够助我上岸，当然了，我对英语的态度要很好，像高三那样好，不然它怎么会帮我呢？英语会不会成为封面上的那把剑带我披荆斩棘呢？如果能的话，我会回来续写我的心情的，加油。

SSRF浅析

CSRF与SSRF

首先来说一下这两个长得这么像的东西，分别是什么，它们有什么区别？

XSS与CSRF

说到CSRF，不得不提一下XSS。从名字上来看，同为跨站攻击，XSS是跨站脚本攻击，CSRF是跨站请求伪造，也就是说，CSRF攻击本不是出自用户之手，而是经过第三方恶意攻击者的处理之后，伪装成了受信任用户的“亲历亲为”。

我们所能见到的大部分网站，都是通过cookie等方式记录、分辨出受信任用户的身份，再予以授权的。所以要伪造用户的正常操作，最好的方法是通过XSS或链接引导等途径，让用户在本机发起自己所不知道的危险请求，使得恶意攻击有机可乘，获取用户cookie等信息，以达到身份伪装目的。看到这里不知您是否有所明白，XSS是实现CSRF的诸多途径中的一条，但并不是唯一的一条。

CSRF与SSRF

从字面上来看，SSRF与CSRF不同的是，它是服务器端发出的请求伪造，而非从用户一端发出的。作为受信任用户，服务器当然不可能做出危害用户的事情。它是一种由攻击者构造形成的，由服务端发起请求的一个安全漏洞。因为它是由服务端发起的，所以它能够请求到与它相连但与外网隔离的内部系统。由于服务端提供了从其他服务器应用获取数据的功能（例如分享等功能）且没有对目标地址做过滤与限制，从而给予了攻击者乘虚而入的机会。比如从指定URL地址获取网页文本内容，加载指定地址的图片，下载资源等等。SSRF是利用存在缺陷的web应用作为代理，攻击远程和本地的服务器。

总结

总的来说，CSRF是服务器端没有对用户提交的数据进行严格的把控，导致攻击者可以利用用户的Cookie信息伪造用户请求发送至服务器。而SSRF是服务器对用户提供的可控URL地址过于信任，没有经过严格检测，导致攻击者可以以此为跳板攻击内网或其他服务器。

如何判断SSRF是否存在

首先要知道出现SSRF的函数基本就这几个file_get_contents()、curl()、fsocksopen()、fopen()，如果获取到源码了，发现源码中存在这些函数，那么大致就可以判断是否有SSRF存在。

如果没有获取到源码，SSRF的入口一般是出现在调用外部资源的地方，比如有个url参数让你传入，或者是在html的输入框中，你可以使用http://、file://、dict://协议读取一下。

PHP学习总结

布尔类型的转换

https://www.php.net/manual/zh/language.types.boolean.php

当转换为 boolean 时，以下值被认为是 FALSE:

  • 布尔值 FALSE 自身
  • 整型值 0 (零)
  • 浮点型值 0.0 (零)
  • 空字符串, 以及字符串 “0”
  • 不包括任何元素的数组
  • NULL，uninitialized
  • 从空标记生成的 SimpleXML 对象

超全局变量

  • $GLOBALS：全局作用域中的全部可用变量
  • $_SERVER：服务器和执行环境的信息
  • $_GET：通过 GET 方法提交的数据
  • $_POST：通过 POST 方法提交的数据
  • $_COOKIE：通过 HTTP Cookies 方式传递给当前脚本的变量所组成的数组
  • $_REQUEST：包含了 $_GET，$_POST 和 $_COOKIE 的所有信息
  • $_FILES：通过 POST 方式上传到服务器的文件数据
  • $_ENV：通过环境方式传递给当前脚本的变量组成的数组
  • $_SESSION：当前脚本可用 SESSION 变量组成的数组

global

PHP中，全局变量在函数内部是不可见的，可通过global或$GLOBALS来访问

  • global实为一个新指针
  • 若有同名，则初始化会与同名变量指向同一数据；若无同名，则初始化为null
  • 不可以在定义global变量的时候直接赋值

static

  • 进入函数的时候静态变量被创建，执行到初始化语句时被初始化
  • 定义静态变量时若不写初始化的值，则默认初始化为 null

运算符

  • $a == ​$b
    • 会进行类型转换
    • 值相等，则返回 TRUE，否则返回 FALSE
  • $a === ​$b
    • 不会进行类型转换
    • 值相等，类型相等，则返回 TRUE，否则返回 FALSE

单引号、双引号、Heredoc

  • 单引号

    • 只支持\‘和\\
    • 速度快

  • 双引号

    • 变量替换
    • 支持更多转义符
    • 速度慢

Heredoc：

https://www.php.net/manual/zh/language.types.string.php

https://wiki.php.net/rfc/flexible_heredoc_nowdoc_syntaxes

  • 变量替换
  • 起始标识符后不能有其它字符
  • 结束标识符缩进不能大于内容中任何一行的缩进

（PHP 7.3 以前）：

  • 结束标识符需顶格
  • 结束标识符分号后不能有其它字符
  • 结束标识符分号后需换行

字符串长度

strlen()

返回值：字符串长度

mb_strlen()

  • $encoding：字符编码。如果省略，则使用内部字符编码

返回值：字符串长度

字符串查找

strpos()

查找字符串中某部分字符串首次出现的位置，区分大小写

  • $haystack：在哪个字符串中查找
  • $needle：要查找的字符串，如果needle不是一个字符串，那么它将被转换为整型并被视为字符的顺序值（ASCII）
  • $offset：指定从 haystack 中的哪个字符开始查找， 如果是负数，搜索会从字符串结尾指定字符数开始。返回的位置数字值仍然相对于 haystack 的起始位置

返回值：找到，返回位置；未找到，返回false

stripos()

查找字符串中某部分字符串首次出现的位置，不区分大小写

返回值：找到，返回位置；未找到，返回false

strrpos()

查找字符串中某部分字符串最后出现的位置，区分大小写

  • $offset：指定从 haystack 中的哪个字符开始查找，如果是负数的偏移量，将会导致查找在字符串结尾处开始的计数位置处结束。返回的位置数字值仍然相对于 haystack 的起始位置

返回值：找到，返回位置；未找到，返回false

strripos()

查找字符串中某部分字符串最后出现的位置，不区分大小写

  • $offset：指定从 haystack 中的哪个字符开始查找，如果是负数的偏移量，将会导致查找在字符串结尾处开始的计数位置处结束。返回的位置数字值仍然相对于 haystack 的起始位置

返回值：找到，返回位置；未找到，返回false

字符串替换

str_replace()

如果 search和replace为数组，那么str_replace()将对subject做二者的映射替换。

如果replace的值的个数少于search的个数，多余的替换将使用空字符串来进行。

如果search是一个数组而replace是一个字符串，那么search中每个元素的替换将始终使用这个字符串

  • $subject：执行函数的数组或字符串。如果subject是一个数组，替换操作将遍历整个subject，返回值也将是一个数组
  • $search：查找的目标值。区分大小写。数组可以指定多个目标
  • $replace：替换值。数组可以被用来指定多重替换
  • $count：替换发生的次数

返回值：返回替换后的数组或字符串

str_ireplace()

$search不区分大小写

substr_replace()

https://www.php.net/manual/zh/function.substr-replace.php

  • $string：执行替换的数组或字符串。如果string是一个数组，替换操作将遍历整个string，返回值也将是一个数组
  • $start：如果start为非负数，替换将从string的start位置开始；如果start为负数，替换将从string的倒数第start个位置开始
  • $length：
    • 正数，表示string中被替换的子字符串的长度
    • 负数，表示待替换的子字符串结尾处距离string末端的字符个数
    • 0，FALSE或NULL，将replacement插入到string的start位置处
    • 不提供此参数，默认到末尾

分隔符

当使用 PCRE 函数的时候，模式需要由分隔符闭合包裹。分隔符可以是任意非字母、数字、反斜线、空白字符。

经常使用的分隔符是正斜线(/)、hash符号(#) 以及取反符号(~)

preg_match()

执行正则表达式匹配

  • $pattern：模式，即正则表达式
  • $subject：输入字符串
  • $matches：将被填充为搜索结果。$matches[0]将包含整个模式匹配到的文本，$matches[1]将包含第一个捕获子组匹配到的文本
  • $flags：
    • 0：默认
    • PREG_OFFSET_CAPTURE：保存偏移量。如果传递了这个标记，会改变填充到matches参数的数组，使其每个元素成为一个由第0个元素是匹配到的字符串，第1个元素是该匹配字符串在输入字符串中的偏移量
  • $offset：指定从哪个位置开始搜索

返回值：返回匹配次数。它的值将是0或1或false。preg_match()在第一次匹配后将会停止搜索。如果发生错误会返回false

preg_match_all()

执行全局正则表达式匹配

  • $flags：
    • 不能同时使用PREG_PATTERN_ORDER和PREG_SET_ORDER
    • PREG_PATTERN_ORDER：结果排序为$matches[0]保存整个模式的所有匹配, $matches[1] 保存第一个子组的所有匹配，以此类推
    • PREG_SET_ORDER：结果排序为$matches[0]包含第一次匹配得到的所有匹配(包含子组)，$matches[1]是包含第二次匹配到的所有匹配(包含子组)的数组，以此类推
    • PREG_OFFSET_CAPTURE：保存偏移量

返回值：返回匹配次数，如果发生错误返回FALSE

  • $flags：

返回值：返回与模式匹配/不匹配的元素组成的数组，键还是原来的键
